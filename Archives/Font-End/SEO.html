When Search Meets Web Usability [SEO] 11月 09日

  在图书馆看到本书《网站搜索设计》,兼顾SEO及可用性的网站设计心得。看着是图灵的，由于想深入了解下SEO什么的，顺手就拿回来了，反正这书个头不大。

  小半个上午就翻完了半本书，很多细节没有细看，毕竟我又不是专门的SEO专员...一直感觉自己学的东西杂而不精，然后就有种啥都没学到的感觉。

  切入正题，When Search Meets Web Usability

SEO:传统上，SEO是一种为整个网站进行设计，文案创作，编写HTML代码、脚本和程序的过程，旨在提高网站或网页吻合特定关键词的几率，提升其在搜索结果列表中的排名。当然了《网站搜索设计》这本书不单纯的只是围绕着WEB搜索引擎优化的，提出了应该除了搜索引擎优化更应该关注用户群体。

链接开发度:(Link Development)，是指指向某个网站的外部第三方链接的数量和质量。所有的主流搜索引擎都利用某个网站的链接广泛度(Link Popularity)来确定在搜索结果中的排名。所以嘛，你经常可以看见有些Blog主天天挨那和别人交换链接，做个友链呢。还时不时的定期清理下无效的链接....好吧，不吐槽了。

网站可用性:个人理解是，并不是说一个网站能够正常能运作，可以被用户使用就完事了，有效性、效率、可学习性，满意度什么的能够提升用户体验的东东才能提升网站可用性，当然后面会提到的A/B测试。

关于WEB搜索引擎的三类查询：

  • 引导型查询

    • 就比如你一时忘记某个网站的网址了，或者不想输入过长的URL，直接搜索吧。有10%~26%的搜索查询属于引导型。

    • 多买些域名，包括公司的名称啊，字母缩写啊，旗下的品牌啊，商标用语。因为有些域名那么相似，总有用户会输错的，网站这样才会较有前瞻意识。
    • 搜索引擎营销---SEM(Search Engine Marketing)
    • 对于引导型查询，根据经验，抱着URL补偿过65个字符，长了人也烦嘛，就比如亚马逊的短连接地址，Z.CN，这真是极好的啊。

  • 信息型查询

    • 为了查找某些方面的信息而进行的搜索，比如我经常百度些how to的问题，以及google看来自stackoverflow的问题的回答。最普遍的搜索引擎查询，占了48%~80%

    • 对于信息型搜索，增加常见的FAQ页面专门解答那些对关键词的回答。
    • 维基百科是一个信息型的网站，与之相对的，在线零售商城则主要属于事物型的网站
    • ①丰富meta标签的内容，因为搜索引擎主要是从title和meta标签以及符合搜索关键字的网页内容来排列搜索结果的，所以这能够增加关键字的吻合概率。一般200-250个字符；②为每一个页面给出专属的meta 标签，这一点我的blog没有做到，个人blog的SEO什么的，I don't care。③在meta和主要内容进行双重曝光。

  • 事物型查询

    • 除了搜索阅读信息外，还行干些什么呢，购物啊，下载啊，这些交互行为就属于事物型查询了。大概有10%~24%的搜索查询属于这块的。

    • A/B Split Testing,A/B测试主要是用于SEO活网络营销通过测试方法来比较对象的(就比如两个不同版本的的网页设计啊，用Google Analytics统计用户体验)，然后选择一个用户体验度好的版本就是咯。
    • 不要假设锁锁着会愿意接受自发地行为，今天登陆江西移动真的是，那么大的焦点图，不就双十一嘛，弹了足足有十秒的样子。好在有关闭的X，不然真心有没有关闭的X的强迫症。

以上，没有全部写出看书的总结，还是觉得这些东西有点虚啊，纸上得来终觉浅。以后关于SEO相关的就放在这个页面咯~~
